📥 Installation

📦 Depuis le dépôt QGIS

  1. Ouvrez QGIS
  2. Menu Extensions → Gérer les extensions
  3. Recherchez "Universal XY Converter"
  4. Cliquez sur Installer

🐙 Depuis GitHub

  1. Téléchargez le ZIP depuis GitHub
  2. QGIS → Extensions → Installer à partir d'un ZIP
  3. Sélectionnez le fichier ZIP

🚀 Utilisation

📍 Onglet 1 : Conversion simple

  1. Sélectionnez le CRS source (ex: WGS 84 degrés)
  2. Sélectionnez le CRS cible (ex: BFTM Burkina Faso)
  3. Choisissez le format (DD, DMS ou Mètres)
  4. Entrez les coordonnées X et Y
  5. Cliquez sur "Convertir →"

📁 Onglet 2 : Traitement batch CSV

  1. Cliquez sur "Parcourir" pour charger votre fichier CSV
  2. Sélectionnez les colonnes X et Y
  3. Choisissez le séparateur (;, virgule, |, tabulation)
  4. Sélectionnez les CRS source et cible
  5. Choisissez le format des coordonnées dans le fichier
  6. Cliquez sur "Lancer la conversion"

🖱️ Onglet 3 : Sélection interactive

  1. Choisissez le système cible pour l'affichage
  2. Sélectionnez le format d'affichage (DD, DMS, Mètres)
  3. Cliquez sur "Activer le sélecteur"
  4. Cliquez sur la carte
  5. Les coordonnées converties s'affichent instantanément

🔧 Définition de CRS personnalisés

Pour utiliser un système de coordonnées qui n'est pas dans la liste, sélectionnez "--- Personnalisé (PROJ) ---" et entrez une chaîne PROJ.

📐 Structure d'une chaîne PROJ

+proj=TYPE +param1=valeur1 +param2=valeur2 +param3=valeur3 +units=m

🌍 Exemples de CRS personnalisés

1. Adindan (ancien système du Burkina Faso)

+proj=longlat +ellps=clrk80 +towgs84=-118,-14,218
💡 Explication : Coordonnées géographiques (degrés) avec ellipsoïde Clarke 1880 et transformation vers WGS84.

2. Adindan UTM 30N (pour données en mètres)

+proj=utm +zone=30 +ellps=clrk80 +towgs84=-118,-14,218 +units=m

3. Lambert 93 (France)

+proj=lcc +lat_1=44 +lat_2=49 +lat_0=46.5 +lon_0=3 +x_0=700000 +y_0=6600000 +ellps=GRS80 +units=m

4. Swiss LV95 (Suisse)

+proj=stere +lat_0=46.9524056 +lon_0=7.43958333 +k=1 +x_0=2600000 +y_0=1200000 +ellps=GRS80 +units=m

5. Projection personnalisée type BFTM

+proj=tmerc +lat_0=0 +lon_0=-1.5 +k=0.9996 +x_0=600000 +y_0=0 +ellps=GRS80 +units=m

6. WGS84 (degrés) vers Adindan (transformation inverse)

+proj=longlat +ellps=clrk80 +towgs84=118,14,-218

7. Clarke 1880 UTM 30N (Afrique du Sud)

+proj=utm +zone=30 +ellps=clrk80 +towgs84=-118,-14,218 +units=m

8. NAD83 (Amérique du Nord)

+proj=longlat +ellps=GRS80 +towgs84=0,0,0,0,0,0,0

9. UTM zone 31N personnalisé

+proj=utm +zone=31 +ellps=WGS84 +units=m
📌 Paramètres PROJ courants :
+proj=longlat : coordonnées géographiques (degrés)
+proj=utm : projection UTM
+proj=tmerc : Mercator transverse
+proj=lcc : Lambert conique conforme
+proj=stere : projection stéréographique
+zone=XX : zone UTM (1-60)
+ellps=GRS80 : ellipsoïde GRS80
+towgs84=dx,dy,dz : transformation vers WGS84
+units=m : unités en mètres

💡 Astuces et bonnes pratiques

📝 Saisie DMS

  • Utilisez °, ', "
  • Directions : N,S,E,W,O
  • Exemple : 12°30'15" N
  • Sans direction, la valeur est positive

🔄 Échanger source/cible

Utilisez le bouton "⟷ Échanger source/cible" pour inverser la conversion.

📋 Copier les résultats

Cliquez sur "Copier les résultats" pour copier les coordonnées converties dans le presse-papier.

📁 Format CSV pour batch

Votre fichier CSV doit être encodé en UTF-8.

id;longitude;latitude;nom
1;12°30'15"N;-1°30'0"W;Ouagadougou

🇧🇫 BFTM - Burkina Faso

  • Ellipsoïde: GRS80
  • Méridien central: 1.5° Ouest
  • Fausse est: 600 000 m
  • Facteur d'échelle: 0.9996

🎯 Notations françaises

  • O = Ouest (West)
  • E = Est (East)
  • N = Nord (North)
  • S = Sud (South)
💡 Astuce : Si vous avez des coordonnées avec des virgules comme séparateur décimal (ex: 12,5042), remplacez les virgules par des points avant conversion.

❌ Résolution des erreurs courantes

🔴 "Can not decode: codec"

📖 Message d'erreur : Can not decode: codec
🔧 Solution :
  1. Ouvrez votre fichier CSV avec Notepad++ ou Visual Studio Code
  2. Vérifiez l'encodage : doit être UTF-8 (sans BOM)
  3. Dans QGIS, allez dans Paramètres → Options → Locale
  4. Vérifiez que le séparateur décimal est le point (.) et non la virgule (,)

🔴 Configuration Windows pour les séparateurs décimaux

📖 Problème : Windows utilise la virgule (,) comme séparateur décimal.
🔧 Solution :
  1. Allez dans Panneau de configuration → Heure et langue → Langue et région → Numéro
  2. Changez le Symbole décimal de , (virgule) à . (point)
  3. Assurez-vous que le Séparateur de liste est ; (point-virgule)
  4. Redémarrez QGIS

🔴 "CRS source invalide" ou "CRS cible invalide"

🔧 Solutions :
  • Vérifiez que le code EPSG est correct (ex: EPSG:4326 pour WGS84)
  • Assurez-vous que la chaîne PROJ personnalisée est correctement formatée
  • Vérifiez que les paramètres comme +zone sont valides

🔴 "Coordonnées invalides" ou format DMS erroné

🔧 Solutions :
  • Utilisez les symboles corrects : ° (Alt+248 ou degrés), ' (apostrophe), " (guillemet)
  • Pour Windows : tapez les symboles via la table des caractères (Win + .)
  • Exemple valide : 12°30'15" N (avec espaces)

🔴 Fichier CSV ignoré (lignes non converties)

🔧 Solutions :
  • Vérifiez l'encodage (doit être UTF-8)
  • Vérifiez le séparateur (;, virgule, |, tabulation)
  • Vérifiez qu'il n'y a pas d'en-têtes mal formés
  • Assurez-vous que les colonnes X et Y sont correctement sélectionnées

🔴 La conversion batch ne produit pas le même résultat que la conversion simple

🔧 Solutions :
  • Vérifiez que le format source (DD/DMS/Mètres) est correctement sélectionné
  • Vérifiez que les CRS source et cible sont identiques
  • Assurez-vous que les données CSV n'ont pas d'espaces invisibles
📌 Bon à savoir : La plupart des erreurs viennent de problèmes d'encodage (UTF-8) ou de séparateurs décimaux (point vs virgule).
